How GNS Auth Works

    

      

        
01

        
User clicks "Sign in with GNS"

        
The website generates a cryptographic challenge (random nonce + origin + timestamp) and dispatches it to the page.

      

      

        
02

        
Extension signs the challenge

        
GNS Vault Chrome extension sees the challenge, signs with the user's Ed25519 private key, and returns the public key + signature.

      

      

        
03

        
Server verifies via Vault API

        
The website sends the public key to vault.gcrumbs.com/v1/verify which returns trust score, breadcrumbs, and badge tier.

      

    

  


  

    
Add to your website

    
10 lines. No OAuth, no cookies, no third-party tracking.

    

      

        

        index.html
      

      
<!-- 1. Add the button -->
<button onclick="signInWithGNS()">Sign in with GNS</button>

<script>
async function signInWithGNS() {
  // 2. Create + dispatch challenge
  dispatchEvent(new CustomEvent('gns-auth-request', {
    detail: {
      nonce: crypto.randomUUID(),
      origin: location.origin,
      timestamp: new Date().toISOString(),
      expiresIn: 300
    }
  }));
}

// 3. Listen for signed response from extension
addEventListener('gns-auth-response', async (e) => {
  const { publicKey, signature } = e.detail;

  // 4. Verify identity via Vault API
  const res = await fetch('https://vault.gcrumbs.com/v1/verify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ public_key: publicKey })
  });
  const id = await res.json();
  console.log(id.human, id.badge_tier);
});
</script>
